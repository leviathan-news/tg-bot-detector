# Local Bot Registry

This directory contains your **local** bot registry — a JSON file of Telegram user IDs flagged by heuristic scoring on your channel.

## No IDs Are Distributed

This package does **not** ship any real Telegram user IDs. The `known_bots.example.json` file contains only placeholder data to demonstrate the format. Each channel operator generates their own registry from their own channel data.

## Generating Your Registry

```bash
# Generate a registry of users scoring >= 4 on your channel
tg-purge registry generate --channel @your_channel --threshold 4 --output registry/known_bots.json

# Add manually flagged IDs
tg-purge registry add --ids-file flagged_ids.txt

# Check a specific user
tg-purge registry check --user-id 123456789
```

## Format

```json
{
  "version": 1,
  "description": "Local bot registry generated by tg-purge",
  "last_updated": "2026-02-28T12:00:00Z",
  "channel": "@your_channel",
  "entries": [
    {
      "user_id": 123456789,
      "score": 5,
      "date_flagged": "2026-02-28",
      "detection_method": "heuristic_scoring",
      "notes": "deleted_account(+5)"
    }
  ]
}
```

### Fields

| Field | Type | Description |
|-------|------|-------------|
| `version` | int | Schema version (currently 1) |
| `description` | string | Human-readable description |
| `last_updated` | string | ISO 8601 timestamp of last update |
| `channel` | string | Channel the registry was generated from |
| `entries[].user_id` | int | Telegram user ID |
| `entries[].score` | int\|null | Heuristic score at time of flagging |
| `entries[].date_flagged` | string | Date the user was flagged |
| `entries[].detection_method` | string | How the user was flagged (e.g., `heuristic_scoring`, `manual_add`) |
| `entries[].notes` | string | Scoring signals or manual notes |

## Privacy Considerations

Registry files contain real Telegram user IDs. These are **not** public information and should be treated with care:

- **Never commit registry files to public repositories** (`.gitignore` already excludes `registry/*.json`)
- **Never share registry files** without understanding the privacy implications
- **Heuristic scores are not ground truth** — false positives are possible at every threshold
- **User IDs are persistent** — they identify real Telegram accounts even if the user changes their username or display name

The `.gitignore` in this repository excludes `registry/*.json` but allows `registry/*.example.json` to preserve the format documentation.
